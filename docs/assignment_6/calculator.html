  • Instructions for using the app
    Key "K"
    Hitting the key "K" will increase the speed of the row & column selection by 250 milliseconds
    Key "J"
    Hitting the key "J" will activate a selection based on the current state of the interface:
    Start
    On start, the row selection will begin, each row will be highlighted based on the starting speed (2000 ms)
    Row Highlighted
    If a row is currently highlighted, the system will switch to iterating over the cells in that row. Once the last cell in the row has been reached, then the interface will return to row selection
    Cell Highlighted
    If a cell is currently highlighted, then the system will activate/select that cell's button

  • Discussion of Test Results:

    There is a clear trade off in speed vs accuracy in using a one-click interface. The faster speed does make it quicker to work through an equation, however there was a higher number of errors. Even with the higher number of errors, the faster speed was still significantly faster (overall), however the regular mistypes led to the application feeling more like a game of luck (similiar to the arcade games where you try to time your button press to when a prize). This made the application feel more frustrating to use. So while the slower speed took more time it wasn't nearly as stressfull/frustrating to use. A more "medium" speed closer to half a second or a full second might provide a more optimal experience.
